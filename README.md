## What this serves

- `.well-known/xrp-ledger.toml` — Domain verification file for XRPL validator attestation

## Setup

This is a GitHub Pages site. The `CNAME` file points to `pft.permanentupperclass.com`.

### DNS

Ensure your DNS has a CNAME record:
```
pft.permanentupperclass.com → P-U-C.github.io
```

### WebSocket Configuration

The dashboard attempts to connect to the validator's WebSocket API. Edit the `CONFIG.wsEndpoints` array in `index.html` to match your node's actual WebSocket port:

```javascript
wsEndpoints: [
  'wss://pft.permanentupperclass.com:51233',
  'wss://pft.permanentupperclass.com:6006',
],
```

### Attestation

Copy the `attestation` value from your existing TOML into `.well-known/xrp-ledger.toml`. This is generated by the `validator-keys` tool and required for domain verification.
