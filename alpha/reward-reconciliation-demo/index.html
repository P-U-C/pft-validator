What is the batch root hash?

It is the SHA-256 over the sorted list of every record’s idempotency key. If any record’s inputs changed, its key changes, and this root changes — so the root is a tamper-evident fingerprint of the whole reconciliation batch.

How this works

Each result is produced by a deterministic reconciliation composer: it compares a historical contribution’s fair value against the total already paid (all partial payments aggregated as a ledger, never last-write-wins) and reports the delta owed, a status, and a machine-readable reason code. Every record carries an idempotency key — a SHA-256 over the exact canonical inputs — so the same inputs always yield the same key, and any change to payment state or policy changes it.

This page re-hashes each idempotency key in your browser from its canonical pre-image and re-derives the batch root, so you can confirm the results are reproducible without trusting any server. The demo data is a synthetic, privacy-safe fixture — no real participants or wallets.
