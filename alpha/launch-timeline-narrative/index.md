## 3. Value accrual — the "contribution → value" loop, stated honestly

What is **observably true today** (all citable):
1. **Work in:** verified contributors (badges: X-metrics for KOLs, GitHub allowlist for core devs, scored expertise for analysts) deliver artifacts through routed tasks with manual verification and evidence requirements.
2. **PFT out:** rewards are paid on-chain per verified deliverable, with observed bands from 3.5 PFT (micro personal tasks) to 35,000 PFT (network build tasks), and review latencies measured in hours.
3. **Integrity enforced:** rewards are protected by delivered tooling — sybil defense, clawback capability, penalty-graded reviews — so a PFT earned reflects work actually verified.
4. **The loop (stated at its defensible strength):** PFT budgets route the network's scarce resources — contributor attention and reach. More verified output → more credible network → more contributors and more demand for work denominated and paid in PFT. That is a **utility-demand case**, and it is the strongest claim this document makes; anything stronger — supply mechanics, sinks, redemption, governance weight — is founder territory **[FOUNDER INPUT 2]**. Attention (cycle-1 board reach figure: 568,266, canonicalized per the content kit) is the newest resource being brought into the same loop.

What this document deliberately does **not** claim: token supply, emission, listing venues, mainnet dates, or price mechanics — **[FOUNDER INPUT: Act-3 flagship facts]**. The arc is built so all of that is optional; the loop above is defensible with zero forward-looking statements.

## 4. Contributor-benefit flow (the funnel KOL content points into)

1. **Arrive** — Discord/X → Task Node signup (minutes).
2. **Verify** — badge path fits the person: KOL (X metrics), developer (GitHub), domain expert (scored task history), QA (verified top-up). Observed: expert badge issued and first task routed within 48 h of engagement.
3. **Receive work** — tasks route automatically with ~24 h accept-by windows; refusing is free and recorded, silence just expires.
4. **Deliver + prove** — public artifact + Discord announcement; evidence requirements stated on the task card.
5. **Get paid** — manual verification → PFT to your XRPL wallet; same-day observed across this operator's own July records for clean evidence (network-wide latency: measure in cycle 1); penalties (not rejections) for weak evidence, with reasons.
6. **Compound** — delivery history feeds routing priority and re-tiering; good operators get more and larger tasks (observed range: first task 3.5 PFT → standing operator averaging 16K+/task).

Every step above is defensible from network records; none requires a roadmap promise.
